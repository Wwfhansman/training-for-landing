# AI 漫剧工作台 — 产品需求文档 (PRD)

## 一、产品概述

### 1.1 产品名称
**Kuro Creative Studio** — AI 漫剧片段全流程工作台

### 1.2 产品定位
面向个人创作者的本地化 AI 漫剧制作工具。将剧本文本转化为 AI 生成的漫画风格视频片段，覆盖「剧本编辑 → AI 分镜 → 首帧图生成 → 视频合成 → 导出」全流程。

### 1.3 目标用户
- 个人漫剧/短视频创作者
- 独立动画工作室
- AI 内容创作爱好者

### 1.4 核心价值
- **一站式**：从文字到视频的完整工作流
- **AI 驱动**：自动化剧本解析、画面描述生成、首帧图生成、视频合成
- **可编辑**：全程支持人工介入调整，AI 结果不满意可随时修改重生成
- **本地运行**：无需云端部署，保护创作隐私

---

## 二、核心工作流

```
剧本输入 → AI 片段划分 → 分镜编辑 → 首帧图生成 → 视频生成 → 导出剪辑
```

### 2.1 流程详解

| 步骤 | 操作 | 说明 |
|------|------|------|
| ① | 输入/粘贴剧本 | 在剧本编辑器中输入或粘贴完整剧本文本 |
| ② | AI 片段划分 | 调用 LLM 分析剧本，自动切分为多个「片段 Fragment」 |
| ③ | 查看/编辑片段 | 右侧面板展示片段列表，支持编辑片段内容和属性 |
| ④ | 进入分镜视图 | 每个片段展开为若干「分镜 Shot」，AI 自动生成 Visual Prompt |
| ⑤ | 生成首帧图 | 为每个分镜生成对应的首帧画面图片 |
| ⑥ | 视频合成 | 将首帧图+片段描述发送给视频大模型，生成 15s 片段视频 |
| ⑦ | 导出 | 下载各片段视频文件，交由外部剪辑软件完成后期 |

## 三、功能模块详细规格

### 3.1 剧本编辑模块

#### 功能列表
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 剧本文本编辑 | P0 | 富文本/纯文本编辑器，支持场景描述、台词、旁白等 |
| 字数统计 | P1 | 实时显示剧本字数 |
| 自动保存 | P1 | 编辑内容自动保存到本地数据库 |
| AI 片段识别 | P0 | 点击按钮，AI 自动将剧本拆分为片段 |
| 片段列表展示 | P0 | 右侧面板展示 AI 划分的片段卡片 |
| 片段手动编辑 | P0 | 支持修改片段内容、类型标签 |
| 重新分析 | P1 | 修改剧本后可重新触发 AI 分析 |

#### 交互说明
- **布局**：左侧 7 列为剧本编辑器，右侧 5 列为 AI 片段面板
- **片段卡片**：显示片段序号、类型标签（环境/互动/紧张等）、内容摘要、「生成分镜」按钮
- **片段卡片左侧**：彩色竖条指示片段类型
  - 紫色 = 环境描写
  - 橙色 = 人物互动
  - 红色 = 紧张冲突

#### AI 片段划分规格
- **输入**：完整剧本文本
- **输出格式**（JSON）：
```json
{
  "fragments": [
    {
      "order": 1,
      "type": "environmental",
      "content": "雨水像密集的箭簇击打在碎裂的玻璃窗上...",
      "shots": [
        {
          "order": 1,
          "visualPrompt": "广角俯拍。繁华的赛博朋克都市霓虹灯光映在潮湿的...",
          "cameraAngle": "wide_angle",
          "mood": "ominous"
        }
      ]
    }
  ]
}
```

### 3.2 分镜编辑模块

#### 功能列表
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 分镜卡片展示 | P0 | 横向排列各分镜 Shot 卡片 |
| Visual Prompt 编辑 | P0 | 每个 Shot 显示并可编辑 AI 生成的画面描述 |
| 生成首帧图 | P0 | 点击按钮为单个 Shot 生成首帧图片 |
| 重新生成 | P0 | 已生成的首帧图可重新生成 |
| 全局风格 Prompt | P1 | 底部全局风格栏，统一调整画风（如「冷调蓝色」「电影胶片质感」） |
| 片段切换 | P0 | 顶部显示当前片段信息，可切换不同片段 |

#### 交互说明
- **布局**：全宽展示当前片段的所有分镜卡片
- **每张卡片**：上部为图片区域（未生成时显示占位），下部为 Visual Prompt 文本 + 操作按钮
- **底部**：全局风格调整栏（带 Apply 按钮）

#### 首帧图生成规格
- **模型**：`gemini-3.1-flash-image-preview`
- **接口**：`POST /v1/chat/completions` with `modalities: ["image", "text"]`
- **输入**：Visual Prompt 文本 + 全局风格 Prompt
- **输出**：Base64 编码图片 → 保存为本地文件
- **宽高比**：默认 `16:9`（1344×768），适配视频

### 3.3 视频渲染模块

#### 功能列表
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 渲染参数设置 | P1 | 视频尺寸、模式（std/pro）、时长等 |
| 生成视频 | P0 | 将首帧图+片段描述提交给视频模型 |
| 生成状态追踪 | P0 | 实时显示各片段视频的生成进度 |
| 视频预览 | P0 | 生成完成后可在线预览视频 |
| 序列时间轴 | P1 | 底部展示所有片段视频缩略图时间轴 |
| 单个视频下载 | P0 | 下载单个片段视频文件 |
| 全部导出 | P1 | 打包下载所有片段视频 |

#### 交互说明
- **布局**：中央为视频预览/合成区，右侧为渲染参数面板，底部为序列时间轴
- **中央区域**：显示首帧→合成中动画→尾帧的三联视图
- **参数面板**：Mode 选择（std/pro）、视频尺寸、时长
- **时间轴**：所有片段视频缩略图横向排列

#### 视频生成规格
- **模型**：`kling-v3-omni`（用户指定，若 API 不支持则回退到 `kling-video-o1`）
- **接口**：`POST /v1/videos`（创建任务）+ `GET /v1/videos/:id`（轮询状态）
- **输入**：首帧图（base64 或 URL） + 片段描述文本
- **输出**：视频文件 URL → 下载保存到本地
- **参数**：
  - `size`：`1920x1080`（默认）
  - `mode`：`std`（标准）/ `pro`（高质量）
  - `seconds`：`5` 或 `10`
- **轮询策略**：每 5 秒轮询一次，超时 10 分钟

## 四、非功能性需求

| 需求 | 规格 |
|------|------|
| 运行环境 | 本地 macOS，Node.js 运行时 |
| 数据存储 | SQLite 本地数据库 + 本地文件系统 |
| 并发控制 | 同一时间最多并行 3 个 AI 任务（避免 API 限流） |
| 错误处理 | 所有 AI 调用需有超时、重试、错误提示机制 |
| 响应式 | 适配 1280px 及以上宽度屏幕 |
