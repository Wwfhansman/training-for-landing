## 点众科技：AI 真人剧 w2d3 任务

### 一、基于 AI 真人剧的创作流程观察与思考

**核心**：技术发展导致门槛降低、创作者基数变大；经营重点从“内容产出”转向“优质差异化”和“成本控制”。

友和、酱油等公司都提到：规模化降本的关键在于行业流程标准化。

#### 传统动漫短剧行业流程标准

建立“三大模块”：`脚本分镜模板` → `画面资产库` → `配音预设`

- 脚本分镜模板：核心，也是最难的部分；需要专业编剧团队完成
- 画面资产库：基础部分；需要专业视觉设计师完成
- 配音预设：辅助部分；需要专业配音演员完成

#### 基于 AI 技术赋能的优化方案

1. 脚本分镜模板
  可以用 AI 辅助编剧团队完成脚本分镜模板：先由 AI 生成初版剧本，再由编剧团队修改迭代；编剧团队既可以是人工，也可以是“专家智能体”。随后继续切分优化为分镜脚本。
2. 画面资产库
  “季播”成为用户留存的关键。可用 AI 辅助视觉设计师构建画面资产库：先生成可复用场景的固定素材，并通过对象存储提供统一的入口/出口，为之后全链路生成提供统一的素材底座。
3. 配音预设
  采用带声音生成能力的高级视频生成模型：不再单独配置音效与配音，直接由模型生成。

#### 工作流流程解析

`idea` → 整体剧本 → 分镜脚本 → 分镜图片生成与固定 → 分镜完整片段生成 → 片段拼接与输出

#### 工具使用分析

文字大模型、图像大模型、视频大模型；固定节点下的 `skill` 与自我维护；工作流整合平台；大模型 API / MaaS 平台。

### 二、具体需求

1. AI 模型集成：不同任务/场景/需求需要不同模型支持，如文本生成、图像生成、视频生成等。
2. 工具链自动化：不同任务在同一个平台下完成，需要自动化工具链支持，并提供可视化的控制流程。

### 三、各节点模型需求

#### 1. 剧本、分镜脚本生成与处理

- 通用强推理/强写作模型：用于剧情大纲、角色设定、分场、对白改写（如 Claude；对中文语境适配更好的中文模型，如 kimi、豆包）
- 结构化输出能力强的模型：把剧本转成标准字段（如“场次、人物、地点、情绪、机位、景别、动作、时长”）（如 Gemini、GPT、kimi）
- 垂类自研模型或工作流：用于“短剧味”“爽点节奏”“工业化分镜拆解”；维护企业内部自用 skills 系统或专用工作流

#### 2. 分镜图片生成

1. 从 0 到 1 的“概念分镜 / 美术方向探索”
  Midjourney：style / omni reference 适合先定世界观和角色视觉语言
   Imagen 4：高质量且快速，适合快速试几十版
   FLUX：适合高质量概念探索；重点在“美感上限和探索效率”
2. 角色一致性的连续分镜
  难点：同一人物在多个景别、角度、表情里仍像同一个人
   适合带参考图或自定义模型路线：Midjourney 的 Omni/Character Reference、Ideogram 的 character reference、Adobe 的 Custom Models，以及 Google 展示的 Nano Banana 2（强调多角色、多张图身份一致性）
   核心不是“画得惊艳”，而是“别漂”
3. 按既定镜头语言去“复现构图”
  若你已具备手绘草图、导演 blocking、真人剧参考机位，就更需要“照着结构来”
   Adobe Structure Reference：把参考图的 outline 和 depth 作为结构约束
   Stability image-to-image：适合拿已有草图继续深化
   OpenAI 公开案例：把粗略草图变成图形元素
   关键在“构图服从度”

#### 3. 空镜、场景等素材图片生成

单张环境图质量和复杂光照：Imagen 4
美术气质探索和世界观 moodboard：Midjourney
固定场景多版本复用：Firefly / FLUX Kontext
接产品流水线批量跑：OpenAI / Google / Stability

#### 4. 视频生成

ora、veo、seedance

#### 4. 小结

可见目前主流一站式漫剧平台的功能重心，都放在：

- 全流程节点的统一接入
- 人物/场景/道具的一致性
- 资产的可复用性

对点众科技来说，在各个节点根据任务差异选择不同模型，也是区别于上述商业化平台的重要功能点。

####  设计自己的工作流（初步方案）
- **文本处理阶段**：
  - 先采用中文语境下性价比更高的 **豆包**
  - 同时考虑 **模型切换** 的扩展能力
- **图片与视频模型选择**：
  - 图片模型：**Gemini、cling**
  - 视频模型：**Cling Omini**
  - 重点探索：如何提升 **人物/场景一致性**
- **存储与素材复用**：
  - 接入七牛云对象存储
  - 图片、视频素材统一采用 **Kodo**
  - 针对后续素材重复调用与视频模型传参做适配
- **产品形态**：
  - 一站式全流程操作
  - 全链路把控各节点产出结果
